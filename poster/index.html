1. Introduction

In the Fair $k$-center problem, we are given a set of points \(S\) in a metric space with distance function \(d\).

The goal is to find a set of centers \( C \subseteq S \) of size \(k\) and an assignment function \( \phi: S \to C \) such that \[ r_{C, \phi}(S) = \max_{x\in S} d(x, \phi(x)) \] is minimized and \[ \beta_\ell \le \frac{ |\{x \in S_\ell : \phi(x) = c\}| }{ |\{x \in S : \phi(x) = c\}| } \le \alpha_\ell \quad \forall c \in C, \ell \in \Gamma \] for fairness parameters \(\alpha_\ell, \beta_\ell\)

2. Our contribution

For an input set \(S\) of doubling dimension \(D\), we provide algorithm with \(3+\varepsilon\) approximation factor and additive violation \(\le 4\Delta + 3 \).

When \(k, |\Gamma|, \varepsilon\), and \(D\) are constant, our algorithms require:

  • Linear time in the input set size in the sequential setting;
  • 2 passes in the streaming setting using memory \(O(\log d_{max} / d_{min})\)
  • 5 round rounds in the MapReduce/MPC setting using memory \(\max\{|S|/p, p\}\), where \(p\) is the number of processors used.

3. Related work

Previous solutions to the \(k\)-center problem under fairness constraints are based on linear programming [1, 2] and have been adapted to the Streaming and MapReduce settings [3].

The scalability of such methods is hindered by the size of the linear program that needs to be solved, which in the MapReduce and Streaming settings [3] depends on the number of processors.
